Overview

Interactive programming with shiny can result in deep nesting of code. A closing bracket or parentheses can either be mismatched or omitted completely. The solution is to style the code consistently. Yes.

Background

The styler package in R is designed to style R code.

The package overview details its main functions: style_pkg(), style_file() and style_text(). These functions are added to the â€œAddinsâ€ menu in Rstudio.

# example
style_text("readr::read_csv('./my_table.csv',col_names=c('a','b'))")
readr::read_csv("./my_table.csv", col_names = c("a", "b"))

The tidyverse style guide is a sensible default for writing R code.

# example tidyverse style
styler::style_text("call( 1)", style = tidyverse_style, scope = "spaces")
call(1)

Space indentation is 2 spaces for tidyverse style. This is problematic when writing deeply nested code and fails to provide enough contrast for bracket matching. While everyone has an opinion, my preference is for code to be written with indentation of 6 spaces.
